# 岩站 GRIP STATION — 你可以自己開一條線的抱石場

> 產業：抱石攀岩館　×　風格：導視系統 Wayfinding
> 全館首創 **拖曳式路線設定器 ＋ 自動 beta 求解**：把岩點拖到牆上，系統當場算難度，並解出一套手腳順序，讓一個具備兩節 IK 骨架的火柴人照著爬給你看。

台北內湖一間（虛構示意）只做抱石的場館。品牌從產業本身長出來：抱石館本來就是一套指標系統——每條路線一個色碼、每個岩區一個編號、每塊牆一組角度。所以整站用「六個岩區＝六個月台」的路網語言來組織，而不是用大標題來介紹自己。

## 頁面

1. `index.html` — **路網圖**。map-first 開場：首屏不是大標題，而是一張可互動的 B1 場館路網圖。滑過／鍵盤聚焦任一岩區，右欄即時換成該區的角度、牆高、在牆路線數、主開線師與難度分佈長條；點擊直接帶著岩區條件跳到路線牌。
2. `wall.html` — **路線牌**。24 條掛牌代表路線的資料庫：岩區／難度區間／取向／開線師／關鍵字五組條件複合篩選＋四種排序，即時計數。每張路線牌的點位圖與難度都由與開線台**同一套演算法**算出，底部「丟進開線台重現這條線」會把路線碼帶進編輯器。
3. `setter.html` — **開線台（核心）**。拖岩點、選角度、即時分級、自動解 beta、火柴人逐步攀爬、路線碼存取、開線單送審與 SVG 條碼。
4. `visit.html` — **入場資訊**。票價表、第一次抱石三步驟、場館規則（六條，配原創警示圖標）、周邊交通示意圖、常見問題。

## 首創說明（登記於 ledger `_first_registry`）

**拖曳式路線設定器 ＋ 自動 beta 求解（Drag-to-Author Route Setter with Auto Beta Solver）**——本站是館內第一個「使用者拖曳出來的東西就是網站的內容，而且網站會回過頭來替它演算並演示」的站。

館內既有的 3D／物理站（ridgeline、compositie-optiek、flicker-house、gluhwerk-glass、jinsheng-glove）都是**由站方預先擺好的場景讓使用者觀看或操控**；本站反過來——**牆是空的，內容由使用者拖出來**，系統再依幾何規則回饋三件事：

1. **難度分級**：依手點種類難度、平均與最大步距、腳點密度、牆面角度算出 V0–V10 與定性評語。
2. **beta 求解**：把手點由低到高排序，依「哪隻手比較低」決定換手順序，依手位重心挑選落腳點，逐步產生完整姿勢序列，並標記超出臂展的動態步。
3. **人體演示**：以兩節反向運動學（IK）由肩到手、由髖到腳解出肘膝位置，配合軀幹鬆弛求解，讓火柴人的姿勢隨路線改變，而不是播放預錄動畫。

拿掉這身指標系統的皮膚之後，本站仍剩下一個別站沒有的東西：**一個會替你的創作算出難度、並示範怎麼完成它的編輯器。**

`prefers-reduced-motion` 下不做姿勢內插，直接切換至該步的靜態姿勢，分解動作清單全程以文字並存；無 JavaScript 時 `<noscript>` 指引使用者前往其他頁面，全站導覽與資訊頁不受影響。
