# 摺染紋樣風 Shibori-Symmetry

> 一套「摺法決定對稱、防染決定圖案」的手工絞染工坊視覺語言。核心信念：**不畫圖，只摺布**——所有紋樣都是對稱群作用在一小塊防染基本域上的解，不是誰描出來的線。任何 AI 讀完本檔即可為任一產業重現同一套風格。

---

## 一、設計哲學

1. **防染思維，不是繪圖思維。** 決定「哪裡不要上色」，剩下的交給染料自己在邊界間走。留白（生成白）是主角，不是背景。畫面永遠是「一大片染色 + 被防染保留的白紋」，不要出現描邊插畫。
2. **一角定全幅。** 任何主視覺都應該看得出「對稱」——放射（雪花）、鏡射網格（板締）、或斜向重複（嵐）。單元小、重複多，靠對稱把小基本域鋪成滿幅。
3. **暖色染缸為底，不是紙白為底。** 整個介面像印在一塊染過的布上：**深茜紅為大面積地色**，內容區才是生成白面板。嚴禁米白紙感當通版底色（那是別種風格的反射動作）。
4. **手工痕跡。** 邊緣要有暈染感（暈邊）、色塊要有微噪點，不要數位般的乾淨硬邊漸層。但暈染是「染料滲開」的organic邊，不是模糊陰影。
5. **職人克制。** 文案是師傅的乾式短句，講工序、講材料、講規矩；不用行銷腔、不用「把 X 變成 Y」句式、不用 EST.19xx 徽章。

## 二、色彩系統

雙色域（duotone）為骨幹：**茜紅 × 生成白**；其餘為染缸色點綴，合計面積小。

| 色 | Hex | 角色 | 建議比例 |
|----|-----|------|----------|
| 茜地 vat | `#5F201C` | 通版底色（頁面 background、footer、nav） | ~46% |
| 茜地暗 | `#4A1815` | nav 底、按鈕陰影、漸層暗部 | 輔助 |
| 生成白 cloth | `#E9DEC6` | 內容面板、卡片、亮字（在深底上） | ~30% |
| 生成白暗 | `#DECFB0` | 面板漸層、次級分隔 | 輔助 |
| 茜紅 dye | `#8E2F2A` | 主染色、強調、主按鈕、標題點綴 | ~10% |
| 墨鐵 ink | `#241C17` | 深面板、文字（在生成白上）、描邊 2px | 結構 |
| 梔子金 gold | `#D9A441` | 唯一亮點色：小標籤、mono 標題、現用態、logo 蕊心 | <5% |
| 藍靛 indigo | `#2C5C7A` | 第二染缸色、絞印藍瓣、色譜藍系 | <4% |

**染料調色盤（供互動選色，真實植物染名）：** 茜草 `#8E2F2A`／蘇枋 `#B0483F`／藍靛 `#2C5C7A`／濃藍 `#1E3F5A`／梔子 `#D9A441`／刈安 `#9AA23C`／紫根 `#6E4A72`／柿澀 `#7A4B24`／鐵媒 `#4A4640`。布色（未染）恆為生成白 `#E9DEC6`。

**規則：** 深底上用生成白與金；生成白面板上用墨鐵文字、茜紅小標。金一站只當「一個亮點」，不要金到處都是。

## 三、字體系統（Google Fonts）

- **標題／品牌：** `Shippori Mincho`（600／800）。日式明朝的craft感，襯線但不奢華，適合工坊。字距 `letter-spacing:.06–.14em`。
- **內文：** `Noto Sans TC`（400／500／700），行高 1.65–1.72。
- **標籤／編號／數據：** `Space Mono`（400／700），全大寫、字距 `.14–.3em`，用於 kicker、mono 讀數、頁碼、留位號。
- 字級：品牌 26／頁標 26–27／H2 20–22／H3 15–16／內文 14.5／標籤 10–11。

三族分工固定：明朝＝標題與紋樣名、無襯線＝說明、等寬＝機器讀數與編號。不要用系統預設字堆疊。

## 四、版面與網格

- **開場 = foldcloth-first：** 首屏不放大標 hero，直接是**染台工作台**（左布面預覽、右控制面板），上方僅一條窄刊頭（logo＋店名＋一句 tagline）。「紋樣在你摺／染之前並不存在」。
- 工作台雙欄：`grid-template-columns: minmax(0,1fr) 360px`；≤900px 疊為單欄。
- 內容區用生成白面板（`border:2px solid #241C17; border-radius:3–4px`），浮在茜地上。**無圓角過大、無模糊陰影卡片**；陰影只用於布面（`box-shadow:0 10px 30px rgba(0,0,0,.45)` 模擬布垂墜）。
- 分節用 `border-top:2px solid #3A1410`，不用留白硬切。
- 布面預覽恆為 1:1 正方（`aspect-ratio:1/1`），置中，最大寬約 440。
- RWD：≤560px 收英文小標、縮 nav 型板寬度；控制面板選項 wrap。

## 五、元件配方

**導覽 nav（型板夾 clamp-board dock，本風格識別）：**
- 固定底部（`position:fixed;bottom:0`），茜地暗底、上緣 2px 墨線。
- 每頁一片「型板」：`width:46px;height:26px` 的生成白小板（`linear-gradient(#E9DEC6,#DECFB0)`＋2px 墨框），板中央一顆金色蝶帽 `.wing`（9px 圓），板背後用 `::before` 畫一根木色夾條（`#6b4a24`）。
- **現用態＝夾緊：** 板 `height` 收到 19px（被壓扁）、`box-shadow:0 0 0 2px #8E2F2A`、下緣 `::after` 長出一條茜紅「染edge」、蝶帽 `.wing` `rotate(140deg)`（旋緊）。語意：這一頁的型板被夾緊了。
- ≤560px：隱英文小標、板縮到 38px。

**按鈕：** 主按鈕茜紅底、生成白字、2px 墨框、`box-shadow:0 3px 0 #4A1815`（實色硬陰影，按下位移）。次要按鈕透明底、生成白／墨框。不要圓潤膠囊鈕。

**選項 chip / opt：** 1.5px 墨框透明底；選中翻墨底、生成白字，小標翻金。

**染料色票：** 30px 圓、2px 墨框；選中 `outline:2px solid #D9A441; outline-offset:2px`。

**面板 h2 小標：** Space Mono、11px、字距 `.22em`、大寫、茜紅色、下 1px 分隔。

**表單：** 生成白面板、白輸入框、墨框；錯誤訊息 `#E86A4A`。三步驟進度條用底線色標示當前步。

**footer：** 茜地、上緣墨線；免責 disclaimer 用 Space Mono 11px、暖灰 `#a98a7a`。

## 六、動效規則

克制、服務工序，不做裝飾性大動作。避開全館過載語彙（通用揭示淡入、數字計數、按壓硬陰影當簽名、dashoffset 描繪）。

- **入缸・解開（招牌動效）：** 按鈕觸發後 canvas 以「由中心向外的圓形 reveal」把滿幅紋樣揭出，`~900ms`，`requestAnimationFrame` 推進 `reveal 0→1`；`prefers-reduced-motion` 下直接設 `reveal=1` 瞬間顯示。這是本站唯一的「表演」，對應真實工序「解開布」。
- **拖曳滑桿：** 即時重繪但降解析度（拖曳 150px、放開 300px），`requestAnimationFrame` 節流。
- **型板夾現用態：** `height`／`transform:rotate` `.18–.3s` 過渡。
- **按鈕：** `:active` `translateY` + 硬陰影塌陷 `.08s`（可用，但不是 signature）。
- 全站 `@media(prefers-reduced-motion:reduce){*{transition:none!important;animation:none!important}}`。

## 七、插畫與圖像風格（shibori-resist 技法）

**唯一圖像來源 = 防染引擎。** 全站不得出現任何外部圖片、任何描邊插畫。所有「圖」都是下列演算：

1. **基本域 openness 函數** `o(u,v) ∈[0,1]`：1＝染料到得了、0＝被型板防染留白。型板紋樣各是一條公式：蜘蛛絞（放射環×放射spoke）、豆絞（格點圓）、縞（餘弦條）、菱格（|u|+|v| lattice）、麻葉（angular×radial）、無地（恆1）。用 `smoothstep` 做暈邊。
2. **摺法對稱映射** `foldMap(fx,fy) → (u,v)`：
   - 雪花 sekka：極座標 `r,θ`；`θ` 對 `π/8` 做三角波折疊 → 二面體 D8 kaleidoscope；`v=clamp(r*2)`。
   - 板締 itajime：`fx*4` 取整判奇偶做鏡射；`fy` 同理 → 4×4 鏡射網格。
   - 嵐 arashi：旋 45°後沿一軸做鏡射週期 → 斜向滑動。
3. **上色：** `amt = clamp(o*dip + noise*0.08)`；`color = mix(生成白, 染料, amt)`；套染再疊第二色。canvas `ImageData` 逐像素、低解析度算完 `drawImage` 放大（保留柔邊）。
4. logo／favicon／圖鑑縮圖／訂單絞印全部同一支引擎（或其 SVG 化身）產出，換種子＝換一幅。

**禁止：** thin-lineart 細線描、emoji、任何 stock 風格插圖。圖必須看得出對稱與防染留白。

## 八、Logo 與 Favicon 設計指南

- **Logo：** 圓形「一枚布樣」。茜地圓角方 → 生成白圓 → 八楔（`rotate(45)` × 8 的 `<use>`）放射防染母題（生成白圓點＋菱片）→ 墨蕊＋金心 → 2px 墨外圈。體現 D8 雪花對稱。
- **Favicon：** 同構縮影，inline SVG data URI 寫在 `<head>`：茜地圓角方＋生成白圓＋四顆茜紅花瓣（十字）＋墨蕊＋金心。務必 URL-encode（`%3C…%3E`）。
- 兩者都應「一眼是絞染布樣」，不是抽象標記。

## 九、Do & Don't

**Do**
- 深茜地通版、生成白面板、圖案看得出對稱與留白。
- 明朝標題＋等寬編號＋無襯線內文三分工。
- 型板夾 nav、實色硬陰影按鈕、暈邊有機。
- 文案講工序材料規矩，人名地址價目具體可信但標明虛構。

**Don't**
- ✗ 米白紙感當通版底色（本風格底是染紅，不是紙白）。
- ✗ 紫藍漸層 hero、置中大標＋兩鈕＋三卡模板。
- ✗ 描邊線稿插畫、emoji icon、外部圖片、模糊陰影卡片。
- ✗ 金色濫用、圓潤膠囊鈕、跑馬燈反射式套用。
- ✗ 圖案看不出對稱（那就不是絞染）。
