> 全館首創 **風場對抗式航行物理**：網站裡有一片會自己演化的風，而風是你的對手——它決定你能不能直線抵達目的地，也決定你最後被判定成哪一級學員。

## 頁面

1. `index.html` — **母港（首頁）**。首屏是滿版的觀音亭西灣海面，canvas 以流線粒子畫出即時風場：陣風帶較亮、陸岬後方有一塊明顯的風影。右上角「今日風況」六項讀數由**當日日期**決定（冬季吹東北季風、夏季吹西南風），每天不同、當天固定。往下是「風不會讓你直走」——一張以真實極曲線畫出的風角圖，解釋迎風死角、最快角度與之字形搶風。
2. `sail.html` — **試航診斷（核心功能）**。在同一套風場裡親自開一趟迎風／順風航線，系統逐秒記錄風角、船速與 VMG，完成後算出效率、換舷損失、死角停滯與風影帶秒數，判定 T／G／H／K 四級並輸出一面診斷旗令與建議課表。附「示範領航」讓電腦先開一趟給你看。
3. `fleet.html` — **船隊與旗語**。21 艘社船的名冊（船型／可借等級／狀態／關鍵字四組條件可疊加篩選，即時計數），以及 26 面國際信號旗的完整手冊——點旗子可以掛上自己的旗繩（最多四面），下方即時翻譯每一面的含義。
4. `club.html` — **課程與入社**。四級課表與費用、出海十條安全規定、會員制度與借船費、基地位置示意圖、六則常見問題。

## 首創說明（登記於 ledger `_first_registry`）

**風場對抗式航行物理（Adversarial Wind-Field Sailing）**——全館第一個「環境模擬場本身就是對手」的站。

館內既有的物理／空間類首創各自解決不同的事：`gluhwerk-glass` 的玻璃物理是**材料**在你手上變形；`jinsheng-glove` 的提線戲偶是**角色**跟著你的游標動；`first-gear` 的鍵盤駕駛是**載具**去到你指定的地方；`grip-station` 是**你畫的內容**被系統回頭演算。這些互動裡，環境都是被動的。

本站相反：**風不服從你，而且它在跟你作對。**

1. **風場是連續演化的場，不是參數**。每一格水面的風向與風速由「基準風 ＋ 六個順風漂移的陣風胞 ＋ 島嶼下風的錐狀風影 ＋ 3–6 分鐘週期的大尺度擺盪」疊加而成，逐幀重算。同樣的水面，五秒後不是同一片風。
2. **極曲線讓「直接前進」在物理上不可能**。船速由真實的極曲線（TWA → 船速占真風速比例）內插決定，風角 28° 以內回傳 0——船頭指向目標時速度為零。要抵達上風的浮標，唯一的辦法是斜切、切到延伸線、換舷、再切回來。這不是難度設計，是這項運動本身的樣子。
3. **對抗的結果被拿來當成評分**。系統逐秒記錄你在這場對抗裡的每一個選擇，最後算出你對風的理解程度。網站的核心功能（技術診斷）與核心互動（操船）是同一件事——你不是先玩再填表，你玩的過程就是答案。

拿掉這身旗語皮膚之後，本站仍剩下兩個別站沒有的東西：**一片會自己演化、且不允許你直線前進的風場**，以及**一個以「你怎麼跟環境對抗」而不是「你怎麼回答問題」來下判斷的診斷器**。

無障礙與備援：`prefers-reduced-motion` 下風場流線改以靜態箭頭網格呈現（風向、強弱仍完整可讀），模擬邏輯不受影響；操舵同時支援方向鍵／WASD、空白鍵自動換舷與觸控左右舵鈕；「示範領航」讓不想自己開的人也能看完整趟並取得診斷。未啟用 JavaScript 時，`<noscript>` 說明航線構成與四項評分標準，課程與費用完整印在 `club.html` 的靜態 HTML 中。全站零外部圖片與音檔、零 3D 函式庫，僅使用 Google Fonts。
