## 核心功能：即時競價（降價鐘搶按）

- 每批 21 秒：上台 3 秒 → 降價 100 格 × 0.15 秒 → 結批。
- 八位模擬承銷人各有**預算、參與率、偏好品項、心理價與 0.18–0.62 秒的反應延遲**；
  誰的心理價先被鐘掃到、誰的手比較快，貨就是誰的。
- 你唯一能看到的情報：**誰在上一批出手、出在哪個價**，以及燈亮琥珀＝那家的手已經在按鈕上。
- 額度會用完；八家的預算也會用完——所以同一場的後段批次中位成交價會比前段低約 8%（實測 0.898 → 0.825 倍參考價）。
- 結果進對帳單：得標批數、支出、**平均溢價（vs 參考價）**、空手率。前兩批是練習批，不扣額度、不列入對帳。

## 插畫技法：花序排列構圖

站上沒有一張花是「畫」出來的。每一張圖都是把該品項在植物學上的**花序型態**寫成排列規則後長出來的：

- **頭狀花序**（大菊、向日葵）：小花以 137.508° 黃金角旋轉、半徑 ∝ √i 堆疊，外圈舌狀花數取 13／21／34。
- **聚繖花序**（滿天星、洋桔梗）：中央花先開，兩側各自遞迴分兩枝，共四階。
- **總狀花序**（百合）：主軸＋由下而上漸短的花梗，左右互生。
- **穗狀花序**（唐菖蒲）：小花無梗，直接著生於主軸。
- **圓錐花序**（文心蘭）：主軸分枝，各分枝自成小總狀。
- **繖形花序**（星辰花）／**單生花**（玫瑰）／**肉穗花序**（火鶴，佛焰苞＋肉穗）。

同一個批號永遠得到同一張圖（批號 → FNV-1a → mulberry32）。
全套圖只用場內四色，**兩批花的差別在排列方式而不是顏色**——這正是驗品組在棚下看的東西。
靜態頁的 24 張圖在建置時就用同一支引擎烘成 SVG，沒有 JavaScript 也看得到。

## 可用性與降級

- 計時是拍賣機制本身，關掉就不是拍賣了，因此不提供暫停；`lots.html` 與 `rules.html` 是內容一樣多的完整靜態頁。
- 前兩批為練習批，可以先按壞兩次再開始。
- 鍵盤 `B` 或 Tab 到按鈕後 Enter／空白鍵即可搶標；成交結果以 `aria-live` 播報。
- `prefers-reduced-motion` 下取消所有轉場與位移，鐘以整格跳動呈現。
- 無 JavaScript 時首頁明示鐘無法運作並導向兩個靜態頁。
- 900px 以下：台車軌道導覽攤平為底部四格，拍賣場改單欄，按鐘鈕滿寬。
