使用指南

01這座圖書館是什麼

Design Skills Center 收藏的不是截圖或靈感牆，而是可執行的風格。每個範例站包含三樣東西：

Demo 網站3 頁以上的完整網站，含原創 Logo、favicon、插畫與動畫。用眼睛驗收風格。
README.md品牌設定、頁面清單、風格關鍵字，快速理解這一站在做什麼。
SKILL.md核心資產。整套視覺語言寫成 AI 可讀的規格書——色票、字體、網格、元件配方、動效參數、Do & Don't。

02選風格 → 下載 SKILL → 交給 AI

流程只有三步：

1. 在藝廊逛 Demo，選定風格。
2. 下載該站的 SKILL.md。
3. 把檔案丟給任何 AI（Claude、其他模型皆可），搭配這樣的指令：

附件是一份網站風格 SKILL 文件。
請完全遵循其中的色彩系統、字體系統、版面規則、元件配方與動效規則，
為「（你的產業／品牌）」建立一個網站，包含（首頁／服務／聯絡）三頁。
內容文案請依品牌重寫，但視覺語言不可偏離 SKILL 規範。
重點：SKILL 定義的是「風格」不是「內容」。同一份 SKILL 可以套用到任何產業——手繪雜誌風的髮廊 SKILL，拿去做咖啡廳、書店一樣成立。

03用種子庫擴充圖書館

到種子庫選一組「產業 × 風格」，複製自動組好的生成指令，交給 AI 產出新的範例站。新站完成後放進 sites/ 目錄，並在藝廊 index.html 加一列索引即可。

04目錄結構

design-skills-center/
├── index.html          ← 藝廊（範例站索引）
├── seeds.html          ← 產業 × 風格種子庫 + 指令生成器
├── guide.html          ← 本頁
├── README.md
└── sites/
    └── （範例站代號）/
        ├── index.html      ← 首頁
        ├── （其他頁面）.html
        ├── assets/logo.svg ← 原創 Logo
        ├── README.md       ← 品牌與頁面說明
        └── SKILL.md        ← 風格規格書（核心資產）

05SKILL.md 撰寫規範

新增範例站時，SKILL.md 必須遵守統一格式，才能保證「任何 AI 讀完即可重現風格」：

Frontmattername（英文 kebab-case 風格代號）、description（英文一句話）
設計哲學這個風格想傳達什麼、適合誰、絕不做什麼
色彩系統全部 hex 色票 + 每色用途與比例
字體系統字體來源、字級 scale、字重與行高規則
版面與網格網格、留白、對齊/不對稱規則、旋轉角度範圍
元件配方導覽列、按鈕、卡片、表單、footer 的具體 CSS 做法
動效規則每種動畫的觸發條件、duration、easing 具體值
插畫與圖像插畫筆觸、描邊、構圖規則
Logo / Favicon造型語言與重製指南
Do & Don't明確清單，含「去AI化」禁令
頁面骨架可直接使用的 HTML 片段

06去AI化守則（全館通用）

本館所有產出都遵守同一套禁令，避免「一眼 AI」的模板感：

✕ 紫藍漸層 hero
✕ 置中大標 + 副標 + 兩顆按鈕 + 三張圓角卡片
✕ emoji 當 icon
✕ 千篇一律的 rounded-2xl + 陰影卡片
✕ Lorem ipsum / AI 腔文案
✓ 不對稱版面、有個性的字體配置、獨特配色
✓ 原創 SVG 插畫與 Logo
✓ 真實可信的品牌文案
