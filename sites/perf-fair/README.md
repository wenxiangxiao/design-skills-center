## 頁面

1. `index.html` — **小全張（首頁）**。首屏沒有大標題，只有一張攤成四十枚郵票的小全張。拖曳平移、滾輪／放大鏡縮放，**縮放層級 L0→L3 決定郵票上有多少資訊**：L0 只有郵區色與面額，L1 長出攤位號與社團名，L2 長出雕刻圖與主打刊物，L3 展開簡介、出攤日與新刊註記。點任一枚即落一枚郵戳，收進郵路手冊。
2. `circles.html` — **攤位名冊**。同一批資料攤平成貼票冊：郵區／出攤日／新刊／價格帶／類型／關鍵字六組條件可疊加篩選，四種排序，即時計數。每張卡片可直接蓋戳，與首頁共用同一本手冊。
3. `route.html` — **郵路規劃（核心功能）**。把蓋過戳的攤位交給系統，依會場走道格線（曼哈頓距離）以**最近鄰起解 ＋ 2-opt 反覆對調路段**收斂出巡場順序，畫在 5F・N 區平面圖上；派一只郵袋沿路走一遍，每到一攤落一枚戳，全部走完產生 `PF-261017-NNN` 完戳證明與依郵路碼雜湊生成的齒孔票根。郵路碼可複製、貼回載入，或以 `?p=A01.B03` 帶進網址。
4. `info.html` — **郵資與須知**。攤位資費表（含單日折扣與首次參加減免）、參展時程、六條觀展須知（原創 SVG 圖標）、五枚場限章樣、常見問題與聯絡窗口。

## 首創說明（登記於 ledger `_first_registry`）

**語意縮放導覽（Semantic Zoom Navigation）**——全館第一個把「縮放」當成主要導覽動作、且**縮放改變的是資訊而不是尺寸**的站。

館內既有的空間類首創（ridgeline 的 Three.js 地形、flicker-house 的 CSS 3D 放映廳、leung-kee 的街景、grip-station 的場館路網）都是「移動視角看同一批內容」；本站不同——**推近鏡頭會讓內容本身長出來**：

1. **L0（×0.45–0.85）**：四十枚郵票只剩郵區色塊與面額數字。這一層要回答的問題是「這場有多大、什麼類型佔多少、我買得起嗎」。
2. **L1（×0.86–1.34）**：攤位號與社團名浮現。回答「有誰在」。
3. **L2（×1.35–1.99）**：雕凹版點刻圖與主打刊物出現，齒孔開始清楚。回答「他們做什麼」。
4. **L3（×2.0 以上）**：郵票本身長高 30px，簡介、出攤日與新刊註記全部展開。回答「值不值得排隊」。

這四層不是同一張圖的四種解析度，而是**四份不同的資訊**：DOM 節點依層級顯隱、雕刻插圖在進入 L2 時才即時運算生成並快取。放大鏡滑桿旁永遠標著目前層級與該層在回答什麼問題，使用者不必猜。

拿掉這身郵政皮膚之後，本站仍剩下兩個別站沒有的東西：**一個把縮放當成資訊層級的目錄**，以及**一個會替你解出最短巡場順序、再派角色替你走一遍的路線規劃器**。

`prefers-reduced-motion` 下郵袋不做逐幀行進、直接跳站落戳，郵戳落章動畫縮為即時顯示；無 WebGL、無 canvas、無外部函式庫。未啟用 JavaScript 時，四十枚郵票的社團名、刊物、價格與簡介已完整印在 HTML 中可直接閱讀，`<noscript>` 另行指路到攤位名冊。
