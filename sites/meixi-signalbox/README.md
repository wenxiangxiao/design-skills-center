# 梅溪號誌樓保存會 MEIXI SIGNAL BOX

**產業**：鐵道號誌樓保存與體驗（觀光鐵道）　**風格**：機械連鎖桿架風 Interlocking-Frame

南投集集・梅溪線 8.4K 的十二桿機械連鎖號誌樓。首頁不是介紹頁，是一台會拒絕你的機器：
你扳桿排進路、取路牌、把兩列對向列車在單線上錯開，扳不動的時候，它會告訴你是哪一支桿子鎖著你。

## 頁面

| 檔案 | 內容 |
|---|---|
| `index.html` | 運轉圖開場（可拖時刻游標讀出各列車位置）＋ 十二桿連鎖台交會作業模擬 |
| `tower.html` | 十二桿連鎖表（可點選查鎖閂關係）、進路表、臂木號誌姿態、**可操作的制動距離諾模圖**、路牌閉塞原理 |
| `booking.html` | 號誌員體驗營預約：日期→場次→職務→人數→試算→驗證→路牌憑證（核心 function） |
| `line.html` | 梅溪線里程坡度、年表、示範運轉時刻表、交通、FAQ、認養一支桿（全靜態） |

## 本站首創（全館第一次）

**安全連鎖邏輯作為統治全站的規則引擎。**
館內既有的互動站，工具都聽使用者的話：gluhwerk 的玻璃隨你變形、shaft-eight 的織機照你排的表織、nodal-luthier 的板子照你刨的厚度回答、foldpoint 的盒子照你拖的角度摺。
本站是第一個**會拒絕使用者、並說明是哪一條規則拒絕了他**的站——而且同一支連鎖引擎同時統治三件事：

1. **核心模擬**：桿與桿之間的鎖閂（鎖錠桿在反位時轉轍器不得動作、號誌現示進行時進路內轉轍器鎖閉、互斥進路二選一、順序鎖讓遠方號誌不可能比主號誌樂觀、佔用中的軌道電路鎖死道岔、目的股道有車則場內號誌不得開放）。
2. **頁面導覽**：右上角的路牌授受器同一時間只有一面路牌在外——這就是單線閉塞規則，你現在持有的那面即你正在看的頁。
3. **預約系統**：一筆預約就是一條進路。同場次不得兼任兩職、不得跨場次重複報名、名額不足即「已鎖閉」，用的是同一套拒絕語彙。

拒絕不是懲罰，是這個產業唯一存在的理由——這也是本站的敘事主軸。

## 版面配方（ledger 欄位）

- **opening**：`schedule-first` 運轉圖開場（首屏即時距圖，無大標 hero；非最近 5 站）
- **nav**：`token-block` 路牌閉塞導覽（新原型）
- **illust**：`nomogram-scale` 諾模刻度構圖（新技法：所有圖像由刻度尺、對準線與讀值標記構成，可讀值）
- **mood**：`vivid-bold`（號誌綠 #0B7A3E 大面積地色＋紅黃訊號色）
- **signature**：連鎖拒絕（refusal-as-interaction）

## 技術

單檔 HTML／CSS／JS，零外部圖片與音檔、零第三方函式庫、零 WebGL。外部資源僅 Google Fonts。
`prefers-reduced-motion` 下所有過渡壓至 0.001ms；連鎖台另備「單步 +20 秒」與「示範操作」，不啟用自動鐘也能取得完整結果。四頁皆有 `<noscript>` 替代管道，line.html 全靜態。
