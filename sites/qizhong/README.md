# 起鐘 Qizhong — 換鐘鳴奏保存會 × 方格記譜風

澹寧堂的一組 ring of six（虛構示意）。整站的核心不是一張漂亮的海報，而是一套**可實際鳴奏的排列引擎**：首頁一打開，六口鐘的順序是 `1·2·3·4·5·6`（rounds），這一頁其餘的「內容」還不存在——你得照 Plain Bob Minor 的換序規則，一排一排把它鳴出來，走一圈再回到 rounds。**每一排順序、每一條 blue line 都不是畫的，是真的排列運算算出來的。**

## 頁面

1. **index.html — 鐘室（起鳴）**：核心互動。選方法（圓鳴／一導／素鳴 60 排）、選你司的鐘、看準落點按空白鍵或你的鐘號把它鳴進正確的空檔。band 中其餘五口鐘準時自動鳴，你要插進它們之間；鳴完回饋你的落點準度與失誤。附「示範自動鳴」自動鳴完整課，供不便操作者取得完整體驗；`prefers-reduced-motion` 下改為不計時的逐排推進模式。
2. **method.html — 換法・編序台**：完整寫出 Plain Bob Minor 的 place notation、blue line（附一張建置時烘成的靜態 SVG，無 JS 亦可見）、叫（bob／single）與「真序」（true touch）的判定。附可操作的**編序台**：每一鉛選素／Bob／Single，系統以真正的排列運算展開，即時告訴你這趟多長、是否 true、是否回到 rounds。
3. **tower.html — 鐘籍・鐘班**：六口鐘的直徑、重量、音名、鑄造年與鐘銘（每口鐘的 SVG 大小與繩長隨鐘號生成）；鐘班名冊；歷來練鳴紀錄可依方法／類型／年份篩選、關鍵字搜尋、點欄名排序。
4. **join.html — 學鳴・入會**：怎麼開始、費用、控鐘安全，與一份**三步驟預約旁聽表**（含 Email／手機驗證，生成 起-YYMMDD-NN 留位號）。

## 首創（登記於 ledger `_first_registry`）

**排列群作為可鳴奏的內容、且換序規則即導覽**：全館第一個以「一個離散有限群 + 其 Cayley 圖上的相鄰對換步行」為核心互動的站。使用者不是調參數看結果，而是在只允許相鄰對換（每口鐘每排至多移動一位）的 method 規則下，逐排把 n 顆鐘的順序從 rounds 換出去、目標是走一條回到 rounds 且不重複任何一排的封閉路徑（a true touch）；系統以真正的排列運算驗證每步合法、整趟是否 true、是否 come round，並把使用者的鳴奏診斷成「節奏（落點）」與「換序（place）」兩個維度。與 droveline（間接控制自主群體）、muli-arbor（逆問題）、kinrin-en（遺傳世代）、shaft-eight（織法矩陣渲染布面）、foldpoint（摺疊運動學）皆不同：那些的互動對象是連續物理量或生物，本站的互動對象是一個排列群與它的生成集，內容（每一排、每一條 method line）就是群元素本身。並延續館內「圖即資料」的作法——blue line、logo、favicon 與 tower 頁六口鐘的圖全部由排列規則生成，不是手繪。
