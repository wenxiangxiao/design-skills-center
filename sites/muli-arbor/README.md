# 木理樹況檢測所 MULI Tree Diagnostics

> 樹木風險評估事務所 × 木質斷層風 Xylem-Tomogram
> 台南・老樹樹幹內部劣化的應力波斷層檢測

## 這個站在做什麼

首頁中央那張斷層圖，在你敲擊之前**並不存在**——不是被遮住、不是還沒載入，是根本還沒有資料。

你要自己決定量測方案：釘幾顆感測器、釘在哪、敲哪幾顆。每敲一顆，這顆與其餘感測器之間就產生射線；
系統沿射線積分真實速度場算出走時，再用 SIRT 代數重建法從這堆走時反演回一張速度場的估計，
一路解出劣化面積佔比、最小殘餘壁厚比 t/R、最弱方位與風險判定。

樹的內部有一個**隱藏真值**（由個體編號雜湊決定，同一株恆為同一個內部）。你反演出來的圖永遠不會等於它——
但你的量測方案越好，它就越接近。切到「揭曉真值」可以當場對答案，看見自己差了幾 m/s。

覆蓋率低於 55% 時，網站會拒絕出具判定，並告訴你灰暗處是「沒量到」而不是「沒問題」。

**首創（全館第一次）**：逆問題作為互動範式——網站藏著一個連使用者也看不到的二維真值場，
使用者不能直接觀察它，只能設計量測幾何；系統由間接量測反演出一個估計，而估計的品質由使用者的量測設計決定，
並且可以當場與真值對照計算誤差。

與館內既有物理／模擬站的差別：`gluhwerk-glass`／`shaft-eight`／`foldpoint` 是使用者操作看得見的對象，
`nodal-luthier` 是使用者操作看不見的參數、但系統做的是**正向**模擬（給定厚度求頻率），
`patchframe` 的隱藏真值是三個純量、且互動是把旋鈕轉向它，
`droveline` 是使用者透過代理間接控制。本站是第一個**反向**求解的站：真值是一整個場，
使用者能改變的只有「怎麼量」，不是「量什麼」。

## 破格出口（可用性保底）

- 反演與判讀的每一項規則都寫在 `method.html`，靜態可讀，無 JS 亦完整。
- 檢測台的每顆感測器同時有畫布上的點與 `S1…Sn` 按鈕，鍵盤與無指標裝置可完整操作。
- 「全部敲一輪」一鍵取得完整結果，不想逐顆操作的人也拿得到判定與報告。
- `prefers-reduced-motion` 下不啟動任何自動動畫（本站本來就沒有自動播放的動畫），過渡壓至 0.001ms，
  邏輯完全不縮減。
- 覆蓋率不足時系統主動拒絕判定並說明理由，不會給出無依據的數字。
- 四頁皆有 `<noscript>` 或靜態替代管道；名錄、收費、判讀方法完全不依賴 JS 即可閱讀。
