# PCB 電路板風格 × Web Audio 聲景合成 技能書

範例站：潮汐訊號 TIDE SIGNAL（基隆港聲音地景計畫）。
本技能書定義「把網站做成一塊會發聲的電路板」的完整規範。

---

## 一、風格定位與適用場景

把頁面視為一塊 PCB：內容是元件，關係是走線，互動是通電。
適用於聲音藝術、電子音樂、科技文化、開源硬體、資料計畫等「訊號」隱喻成立的主題。
不適用於溫暖手感、有機食品、兒童等題材——基板是冷的，這是它的性格。
核心原則：**功能決定圖形**。每條走線必須真的連接兩個有意義的節點；禁止純裝飾的假電路。

## 二、色彩系統

| Token | 值 | 用途 |
|-------|----|------|
| `--bg` | `#0a120d` | 基板：墨綠近黑（dark-moody 底） |
| `--bg2` | `#080f0a` | 更深一層（頁尾、首屏底） |
| `--panel` | `#0d1811` | 模組卡片底 |
| `--copper` | `#c9803a` | 銅箔：走線、邊框、強調 |
| `--hi` | `#eda65a` | 通電銅箔：亮起的走線、脈衝、波形 |
| `--pad` | `#d8a35c` | 焊點金 |
| `--silk` | `#dde4da` | 絲印白：正文、標號 |
| `--muted` | `#8ba394` | 灰綠：輔助文字 |

規則：暗銅（`rgba(148,106,58,.5)`）＝斷電走線；亮銅＝通電走線。兩態必須並存，開關才有意義。
**禁止紫藍漸層**；發光一律用銅色系 `shadowBlur`／柔性 box-shadow，禁按壓硬陰影。

## 三、字體與絲印規範

- 等寬：IBM Plex Mono（或 Space Mono）——標號、代碼、座標、數據、導覽。
- 中文：Noto Sans TC，內文 300、標題 500–700，字距 `.1em–.2em` 拉開如絲印。
- 絲印語彙：元件標號（`U1`、`R12`、`C7`、`TP3`）、腳位名（`GND`、`3V3`）、修訂號（`REV 2.6`）、
  中英並置（`SIG-MIX 混音母線`）。標號永遠大寫、字距寬、10–11px。
- 文案聲線：神秘寡言。短句、少形容詞、允許一行詩（「聲音不留檔。留下的是電路。」），不解釋太多。

## 四、版面與網格

- 板框：頁面或首屏外緣 1px 淡絲印框＋四角定位孔（雙環圓）。
- 元件即卡片：每張卡左上或對角放小焊點記號（`::before` 圓環），如板上元件的極性標記。
- 中央母線思維：首屏以「中央 IC＋放射狀模組」構圖，資訊層級＝電路層級。
- 密度中等：留出基板空地，散置過孔與絲印標號當呼吸點，避免塞滿。
- 佈局禁忌：置中三卡片、對稱網格陳列。改用非對稱的埠（port）列、通道條（strip）列。

## 五、走線、焊點與過孔繪製規則

- 走線只走水平、垂直與 45°，轉角用倒角（chamfer 12–14px），單寬 3px——這是 PCB 的文法。
- 演算法：Manhattan 路徑（H→V 或 V→H）→ 過濾零長段 → 轉角倒角；起訖端各一枚焊點。
- 焊點＝雙環圓墊：外環 2.5px 銅描邊、內心 1.8px 實心、中央基板色鑽孔。
- 過孔＝小號焊點（r 2.4），隨機散置但避開 keep-out 區（模組、IC、文案區域）。
- 用種子隨機（mulberry32）生成裝飾，同一 seed 每次載入相同——電路板不該每次重排。

## 六、元件語彙

- **IC**：圓角矩形、兩側腳位短矩形、第一腳圓點標記、型號絲印（如 `KHM-2380`）、內嵌波形視窗。
- **模組**：HTML `<button>` 疊在 canvas 上（可聚焦、`aria-pressed`），內含標號列＋中文名＋LED 狀態點。
- **LED**：8px 圓點，滅＝`#38412f`，亮＝`--hi` 加 `box-shadow` 光暈，只表示真實狀態。
- **排針選單**：漢堡鈕三橫線即三支 pin；全螢幕選單＝掀開機殼的第二層板（`LAYER 02` 絲印＋淡網格底）。
- **埠**：頁面間連結寫成 `PORT J2`、`PORT J3`，頁尾聯絡區寫成 `J4 · I/O`。

## 七、動效規則與 Web Audio 合成配方

動效總則:動的東西必須是「訊號」。走線脈衝、波形、LED——除此之外一律靜止。
脈衝：亮起走線上以 `speed≈120px/s、spacing≈110px` 的光點沿路徑流動，半徑與透明度綁定該聲道即時 RMS。

**母線**：`各聲道 → GainNode(音量^1.4) → AnalyserNode → master GainNode(總音量^1.6×0.9) → mute GainNode → DynamicsCompressor(threshold −18, ratio 4) → destination`。
預設總音量 45/100（中低）。AudioContext 必須在使用者點擊（「接上訊號」）後建立／resume。

**五聲道節點鏈（全部無音訊檔）：**

1. **雨 U1**＝(a) 白噪音 BufferSource(loop) → Bandpass(5200Hz, Q.7) → hiss Gain（隨密度 .10–.32）；
   (b) 雨滴排程器（setInterval 120ms 前瞻 0.3s）：每滴＝噪音短段(0.14s) → Bandpass(2200–7000Hz, Q5) → Gain 包絡（4ms 起音、50–110ms 指數衰減）。參數「雨滴密度」＝每秒 2–40 滴。
2. **浪 U2**＝白噪音 → Lowpass(420Hz, Q.8) → Gain(.42)；LFO1 sine(1/週期) ×.30 調振幅、×240 掃 Lowpass 頻率；LFO2 sine(0.37/週期) ×.13 疊不規則感。參數「湧浪週期」＝4–14 秒。
3. **霧笛 U3**＝Saw(84Hz)＋Saw(84.7Hz)＋Sub Sine(42Hz×.6) → Gain(.45) → Lowpass(300Hz, Q1) → blast Gain 包絡（`setTargetAtTime` 起音 τ.28、2.2s 後釋音 τ.4）；vibrato sine 4.3Hz×1.3 調兩鋸齒頻率。參數「鳴放間隔」＝7–26 秒，`setTimeout` 鏈排程。
4. **廟口低語 U4**＝兩路白噪音 → Bandpass(游走 230–1070Hz, Q2.2) → Gain 游走；每 350ms 以 `setTargetAtTime` 隨機游走頻率與音量（模擬人聲霧）；偶發鐘聲＝Sine(f)＋Sine(2.76f) 指數衰減 2.6s，機率隨密度。參數「人聲密度」。
5. **船機引擎 U5**＝Saw(38–78Hz) → Lowpass(170Hz, Q1.1) → Gain(.55)；chug sine(4–13Hz)×.35 做 AM 律動；jitter sine(.5Hz)×1.6 微晃基頻；另白噪音 → Lowpass(110Hz) → Gain(.3) 轟鳴。參數「引擎轉速」同步基頻與律動。

每聲道掛一個 `AnalyserNode(fftSize 512)`，供示波器與走線脈衝亮度取 RMS；母線波形顯示於中央 IC 視窗。

## 八、互動與功能模式

- **模組即開關**：點板上模組＝開關聲道，未連線時允許預先切換並提示先「接上訊號」。
- **配置器**：每聲道一條 strip（開關＋示波器＋音量推桿＋一個變化參數推桿），推桿以 `setTargetAtTime` 平滑作用，禁突變爆音。
- **配方代碼**：狀態可序列化——`TS-` + 每聲道［字母＋兩位音量＋兩位參數］；音量 00＝關閉。提供一鍵複製與貼入重現、田野配方預設 chip。
- **音訊塢**：固定底部，常駐總音量／靜音／停止所有聲音／連線狀態 LED。

## 九、RWD 與無障礙降級

- ≤560px：配置器 strips 改單欄直向堆疊；≤620px：電路總圖改直式佈局（模組上下分列、IC 居中），文案區移出畫布。
- `prefers-reduced-motion`：脈衝停止移動（走線仍亮）、canvas 改 400ms 間隔更新、選單與過場零動畫；「停止所有聲音」永遠可及。
- 模組與開關全部用真 `<button>`＋`aria-pressed`；滑桿有 `aria-label`；代碼訊息用 `role="status"`；提供 skip link。
- Canvas 元素附 `aria-label` 描述內容；純裝飾 canvas 標 `aria-hidden`。

## 十、禁忌清單與品質檢核

**禁**：音訊檔案（mp3/wav/ogg 一律不准）、autoplay 未經點擊、跑馬燈、`EST.19xx`、老街屋敘事、
揭示進場（reveal-on-scroll 主打）、數字滾動計數、按壓硬陰影、dashoffset 描線主打、
細線裝飾線描、剪紙、半調網點、guilloché 紋、紫藍漸層、置中三卡片、Lorem ipsum、emoji icon、外部圖片。

**檢核**：
1. 斷網（僅字體快取）能否完整發聲？——必須能。
2. 每條走線是否連接兩個真實節點？
3. 關掉聲道，走線是否確實轉暗、脈衝消失？
4. 預設音量是否中低、首次發聲是否不刺耳？
5. reduced-motion 下是否仍可完整操作並停止所有聲音？
6. 360px 寬度下配置器是否可用、代碼是否可複製？
