# 溪頂鄉調解委員會 XIDING TOWNSHIP MEDIATION COMMITTEE

**產業 × 風格：鄉鎮市調解委員會 × 筆錄速記風 Transcript-Steno**

一間鄉公所二樓的調解室，被做成一個你可以坐下來的網站。首頁不是宣傳，是一份民國八十九年的舊筆錄——它從頭到尾都可以讀，只有中間一格是空的：那天下午，有一句話沒有被說出來。網站要做的事，就是把那一格重新打開。

在調解室裡，**你唯一能做的事是說話**。你不能拖動任何金額、不能替任何人做決定、不能看見他們心裡的數字——只看得到他們的手在做什麼。你說出口的每一句話，會即時被速記引擎轉譯成一條連筆線，堆成這一場的速記卷。那些線就是本站唯一的圖像。

## 核心功能（function 維度）

**協商對話（選句 → 兩造隱藏狀態變化 → 調解成立或不成立）**——全館新功能類型。

- 三件案子（機車擦撞／樓上滲漏／界址與芒果樹），兩造各有姓名、年齡、職業、立場與開場陳述。
- 20 句可選發言，分十類：安撫、探詢、回應情緒、整理爭點、程序告知、施壓、沉默、促成致意、援引法條、背對背。每回合從池中依當下狀態抽 4 句，同一句三回合內不重複出現。
- 每一造有 **怒氣** 與 **信任** 兩個隱藏狀態，**全程不以數字顯示**，只翻譯成行為線索（「把手機翻面蓋在桌上。」「椅子往後拖了半步，聲音很大。」）。
- 效果依當下狀態而變：安撫在對方生氣時最有效、沉默在對方平靜時反而扣分、程序告知對冷靜的人加分對氣頭上的人加怒、施壓在信任 >62 時換來 9% 讓步否則把人推向門口。
- 怒氣達 90 當事人起身離席，當場結束，沒有補救。
- **背對背調解**：把一造請到室外單獨談，對在席者效果 ×1.35，室外者每回合怒氣微升。
- 提案是一個包裹：金額 ＋ 分三期給付／書面致意／互不追究其餘請求。條件會改變兩造的可接受範圍，常比多加幾千元有效。
- 結果產出調解書（含給付方式、案號、以速記線繪成的三方簽名）或調解不成立證明書，並指出**本場的轉折點**——信任合計上升最多的那一句。

## 首創（全館 `_first_registry` 新登記）

**對話作為互動範式，且對話本身就是本站的美術資產。**

1. 全館第一個核心互動是「說話」的站：沒有模擬引擎、沒有量表、沒有可拖曳的物理量。使用者不能操作任何數值，只能從有限的句子裡挑一句說出口。
2. **可接受範圍（ZOPA）不是預設的，是被談出來的。** 兩造坐下時 `floorA > ceilB`，交集為負——這時候提任何數字都會被拒絕，而且每次被拒絕都讓兩人更確信委員站在對方那邊。`floorA = (基準 − 信任×係數) × 條件乘數`、`ceilB = (基準 + 信任×係數) × 條件乘數`，兩條線要先靠情緒工作往中間移動，數字才有可能成立。這條規則完整寫在 `method.html` 上，不藏。
3. **使用者說出口的每一句話，即時成為本站的插圖。** 速記引擎把該句以 FNV-1a 逐字雜湊成八種筆畫原語的連筆線；全站沒有任何一張傳統插圖，速記卷、案卷縮圖、要領卡片、調解書上的三方簽名全部出自同一支引擎。同一句話恆得同一條線。
