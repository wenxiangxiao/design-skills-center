## 首創（全館第一次）

**排隊競用（queue-contention）作為互動範式。** 使用者被序列化進一條與模擬他人爭用有限資源的 FIFO 佇列。 所有其他候位客人——到店時刻、人數、耐性、是否棄號、用餐時長、各自點什麼菜——都是 `f(當日種子, 現在時刻)` 的純函數：無後端、可重現、且會隨真實時段呈現不同的用餐尖峰（11:50 與 13:20 的現場不一樣）。使用者抽號後即被編入這條隊伍，名次因「前方入座／棄號／大桌卡位」即時變動；招牌菜的售罄時刻不是寫死的，而是被這群模擬客人吃出來的（份數合計＜尖峰需求，故兩三味必在午市過半後見底）。

與既有站的差別：`nightspin`（自助洗衣店）的『偽即時多人共同在場』是彼此獨立、平行運轉的機台之非同步在場痕跡——沒有排序、不把使用者插進序列、也沒有座位爭用與翻桌；本站是單一有限座位的**序列競用與翻桌流轉**，使用者被編入 FIFO 隊伍、名次隨前方入座／棄號浮動，且共用資源（座位與菜）被同一群人此消彼長地耗掉。`beitun-bell` 是你 vs 對手搶同一次按鍵的對抗賽局，本站無輸贏、不與人搶按，只是排隊；`droveline` 是你指揮代理去影響 boids，本站你完全不能左右任何一個行人。
