# 鐵窗花風 Iron-Grille ／ iron-grille

> 一套從臺灣老街屋鐵窗花長出來的視覺語言。畫面上的每一塊圖都不是「畫的線」，
> 而是一面焊得起來的窗：黑鐵桿承力、硬邊花面填色、焊點收頭。桿與桿之間有規矩——
> 對得上才是鄰居。任何產業都能用，只要你願意讓版面像一面窗一樣，每一塊都接得住隔壁那塊。

---

## 一、設計哲學

**圖是焊出來的，不是描出來的。**

一般網頁的裝飾線只有「有」跟「無」；鐵窗花的線有身分——它是承力的桿，抵到邊要接得到隔壁的桿。
這套風格把這件事整個搬進版面：所有插圖、logo、favicon、甚至受理單上的小印章，都是同一種東西——
**一面由母題格拼成、每條縫都對得上的鐵窗**。三條原則：

1. **底是一面漆過的老牆，不是白紙。** 地色用高飽和的中間調（湖水綠、天井藍、硃紅、磨石灰），
   帶一層磨石子般的細點斑。花面（象牙、硃紅、鵝黃）是嵌在鐵格裡的填色，不是卡片背景。
   這是全套風格最反 AI 的一手：vivid-bold 通常配深底，本風格偏要**亮牆為地**。
2. **零圓角、零模糊陰影。** 鐵是硬的。所有轉角都是尖角，立體感來自硬邊色塊的鄰接與 3–4px 的實線描邊，
   不來自 box-shadow。唯一的圓是焊點（實心圓）與花心。
3. **鐵桿一律粗、一律黑、一律接得住。** 主桿 6–9px、內桿 4–5px，全部同一種鐵墨色。
   一根桿抵到格子邊，隔壁格子的那條邊必須有桿來接——這條規矩決定了整面窗的樣子。

**不要做的事**：不要把它做成「幾何裝飾風」或「Memphis 撞色」的變體。那些是把形狀灑上去；
本風格的每一塊都必須是一面**焊得起來的窗**——桿要連續、對稱要收得回來、花面是被鐵格圍出來的負空間。
把鐵桿改細、改成裝飾線、或讓花面自由漂浮不被鐵格圍住，這套風格就死了。

## 二、色彩系統

| 名稱 | hex | 用途 | 面積比 |
| --- | --- | --- | --- |
| 湖水綠牆 | `#2FA8A0` | 全站地色（老屋漆牆） | ~46% |
| 牆陰 | `#268C85` | 磨石子點斑、窗背板陰影 | ~4% |
| 鐵墨 | `#15201E` | 鐵桿、外框、頁首、導覽、深色卡 | ~20% |
| 象牙花面 | `#F2ECD9` | 花面、卡片與內文底 | ~26% |
| 硃紅 | `#E8452F` | 花面撞色、現用態、主要操作 | ~6% |
| 鵝黃 | `#F5C518` | 焊點、卷號、強調小色塊 | ~3% |
| 天井藍 | `#3E77B8` | 花面第三色（點綴） | — |

**磨石子點斑**（地色上的細顆粒質感）：以多層 `radial-gradient` 疊在牆色上，每層一種小色點：

```css
background:#2FA8A0;
background-image:
  radial-gradient(circle at 12% 22%,rgba(232,69,47,.16) 0 2px,transparent 3px),
  radial-gradient(circle at 74% 12%,rgba(245,197,24,.18) 0 2px,transparent 3px),
  radial-gradient(circle at 40% 66%,rgba(242,236,217,.22) 0 2px,transparent 3px),
  radial-gradient(circle at 88% 74%,rgba(62,119,184,.16) 0 2px,transparent 3px);
background-size:180px 180px,150px 150px,210px 210px,240px 240px;
```

**換牆換色**：整套可平移到天井藍 `#3E77B8`、硃紅牆 `#C6402E`、磨石灰 `#8C8A80`、粉牆黛 `#E5D9C3`——
牆色換、鐵墨與花面三色隨之換，但「亮飽和牆＋黑鐵格＋象牙花面＋一到兩個撞色」的結構不動。

| 角色 | 字體 | 用法 |

## 四、版面與網格

- **外框思維**：每個內容區塊想成一面窗——3px 鐵墨實線外框、直角、無圓角。深色資訊塊用鐵墨底、象牙字。
- **導覽是一根豎櫺（mullion-rail）**：桌機左側固定一根 74px 寬的鐵墨豎桿，頁面各為一片鑄字牌（直排中文＋mono 英文小標），
  現用頁那片外推 6px、翻成硃紅、右緣長出一個三角榫。`≤900px` 落為底部橫列 dock。頁尾另備完整文字連結保底。
- **主體留白中等**：不是極簡大留白，也不是資訊轟炸；讓鐵格的密與牆的空形成節奏。
- **旋轉/傾斜**：不用。鐵窗是正的，所有元素水平垂直對齊，力量來自對稱與硬邊，不來自傾斜。

## 五、鐵窗花格構引擎（本風格的圖像核心・grille-lattice）

**所有圖像都是同一支引擎解出來的一面窗，沒有一張是描外形畫的。** 這是本風格與「幾何裝飾」最根本的分界。

### 5.1 母題與插口型

把每一格窗看成一張牌，四個邊各有一個**插口型**，描述這條邊上鐵桿的位置：

- `c` 中桿——桿抵在邊的正中央（直櫺、十字、菱格）。
- `x` 角桿——兩根斜桿抵在邊的兩端角落（斜十字、寶石眼）。
- `o` 空邊——這條邊沒有桿，兩片花面直接對接（留白、透風）。
- 外框邊——窗的最外圈，抵到這裡的桿全部焊進外框，不受相鄰限制。

母題（unit 100×100）以「線段（鐵桿）＋多邊形（花面，帶役色索引）＋焊點」定義，例如十字 `[c,c,c,c]`、
斜十字 `[x,x,x,x]`、直櫺 `[c,o,c,o]`、菱格、寶石眼、四瓣花、放射旭日、素面 `[o,o,o,o]`。
每個母題自動生成 4 個 90° 旋轉變體（插口與幾何一起轉），約 40 種變體。

### 5.2 相鄰規則（唯一的硬規矩）

```
左格.右邊 == 右格.左邊    上格.下邊 == 下格.上邊
```

中桿只能接中桿、角桿只能接角桿、空邊只能接空邊。這一條就是「焊得起來」的定義。

### 5.3 波函數塌縮（WFC）

1. **初始**：每一格保有全部變體（依牆色與母題組過濾）。
2. **觀察**：找剩餘選項最少的格（最接近被逼到唯一解），依權重擲骰塌縮成單一母題。
3. **傳播**：沿四縫刪除鄰居中「facing 邊無法與本格任一選項相容」的變體；被改動的鄰居再傳播出去，直到穩定。
4. **收斂**：重複到每格只剩一個。撞牆（某格歸零）就換種子重跑（上限 40 次），仍不行則以整面十字保底。
5. **釘選＝約束**：使用者釘住某格＝在觀察前替它定案，約束從那裡開始傳播。這是核心互動：**下條件，不畫圖**。
6. **對稱**：四方對稱時只解左上象限再鏡射（插口型左右/上下對稱故鏡射恆合法），保證收得回來。

### 5.4 渲染分層（由底到頂）

牆底 → 花面色塊（先鋪）→ 外框粗實線 → 鐵桿（`stroke-linecap:round`）→ 焊點實心圓。
`box-shadow:0 0 0 4px 牆陰` 讓窗浮在牆上。決定論：`FNV-1a(seed) → mulberry32`，同種子恆得同窗，支援 `?w=` 分享與靜態烘焙。

### 5.5 一圖多用

同一支引擎輸出：主視覺即時窗、圖鑑縮圖（建置時烘成靜態 SVG）、logo 標記（3×3 對稱窗）、
favicon（單母題或 2×2）、每張受理單的「窗籍印」（由單號雜湊種子生成的 3×3 微窗）。全站零外部圖片。

## 六、元件配方

- **按鈕**：2px 鐵墨框、直角。實心＝鐵墨底象牙字，hover 翻硃紅白字（`transition:.13s`）。
  `aria-pressed=true` 態翻硃紅。段落切換（seg）＝一排這種按鈕。
- **母題筆（brush）**：一排 42px 小方鈕，每顆內嵌一張該母題的迷你窗 SVG；選中描硃紅框。橡皮＝一個紅 ✕。
- **配色片（palette swatch）**：40px 方塊，以 `linear-gradient` 切三段（牆／撞色／花面），選中描鵝黃外框。
- **讀數表（BOM）**：鐵墨底、象牙字、鵝黃大數字（DM Mono），2 欄格陣、1px 黑縫分格。
- **卡片**：象牙底 3px 鐵墨框；深色版鐵墨底。無圓角、無陰影。
- **表單**：白底輸入框 2px 鐵墨框、focus 3px 硃紅 outline；label 為 mono 小型大寫；錯誤訊息暗紅 mono。
- **收據/受理單**：鐵墨底象牙字、鵝黃單號，右附窗籍印。

## 七、動效規則

會動的只有**使用者操作與塌縮重繪**，沒有自動播放。

| 動效 | 觸發 | duration / easing |
| --- | --- | --- |
| 按鈕換色 | hover / 選中 | 130ms ease |
| 導覽鑄字牌外推 | hover / 現用頁 | 160ms ease |
| 窗即時重焊 | 改參數 / 釘選 / 重焊鈕 | 同步重繪（無過渡＝新資料而非動畫） |
| 讀數/報價更新 | 參數變動 | 同步（不做數字滾動計數） |

**去 AI 化自查**：不做揭示淡入當主打、不做數字滾動計數、不做按壓硬陰影、不做 stroke-dashoffset 描繪、不放跑馬燈。
`prefers-reduced-motion` 下所有 `transition` 歸零；因無自動動畫，故無任何邏輯被停用。

## 八、插畫與圖像風格（grille-lattice）

見第五章。要點：**需要「圖」的地方放一面窗**（大尺寸即時塌縮或靜態烘焙），**需要「圖示」的地方放單一母題**
（一格窗），**需要「裝飾」的地方放焊點、格線或色塊**。永遠不要引入描外形的線稿插圖、照片、emoji——
一旦出現「畫出來的形」，本風格的內在一致性就破功。花面配色只在牆的那一套三色裡選，兩面窗的差別來自母題排列而非色相。

## 九、Logo 與 Favicon 設計指南

- **Logo**：左標記＝一面 3×3 四方對稱鐵窗（外框＋十字主桿＋四角斜桿＋鵝黃焊心），右字標＝
  Big Shoulders 800 的英文名 + Noto Sans TC 700 的中文名 + 一條硃紅豎線隔開的兩行小字說明。整體置於牆色方塊上。
- **Favicon**（32×32 inline SVG data URI）：牆色底＋兩塊象牙花面＋硃紅斜桿＋鐵墨十字＋鵝黃焊心。
  務必在 32px 讀得出「一面窗」——桿要夠粗（2–3px）、只保留最少母題，不要塞滿。

## 十、Do & Don't

**Do**

- 亮飽和牆為地色、黑鐵格、象牙花面、一到兩個撞色。
- 每一張圖都是一面焊得起來的窗——桿連續、花面被鐵格圍住、有焊點。
- 直角、粗實線、硬邊；數字用 mono 且帶單位。
- 導覽、logo、favicon、印章共用同一支格構引擎。
- 文案從產業長出（普查、放樣、複丈、6 分方管、焊點、插口、複製重製）。

**Don't（含去 AI 化禁令）**

- ✗ 紫藍漸層 hero、置中大標＋兩顆按鈕＋三張圓角卡片模板。
- ✗ 把鐵桿當裝飾細線、讓花面自由漂浮不被鐵格圍住。
- ✗ 圓角、模糊陰影、emoji icon、描外形的線稿插圖、任何外部圖片。
- ✗ vivid-bold 配深底（本風格刻意用亮牆）；米白紙感底。
- ✗ 揭示淡入當 signature、數字滾動計數、跑馬燈、自動輪播。
- ✗ 把「幾何圖案」隨機灑上去而不受相鄰焊接規則約束——那不是鐵窗花，是貼圖。
